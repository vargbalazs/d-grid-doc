Grouping templates

The grid provides built-in group templates which help you to customize the content of the group header and footer cells. The available group templates are:

  • Group header template (DGridGroupHeaderTemplateDirective)
  • Group header column template (DGridGroupHeaderColumnTemplateDirective)
  • Group footer template (DGridGroupFooterTemplateDirective)
  • Group footer column template (DGridGroupFooterColumnTemplateDirective)

All templates expose the group, column and aggregate interfaces to help customize the templates.

As per default, the header template looks like this: 'group.field: group.key (group.rows.length)'. The footer templates don't have any default format.

Group header template

The DGridGroupHeaderTemplateDirective is evaluated for the grouped column only. The header template helps you to customize the default group value of a certain field. The group header template is rendered at the beginning of the header row/in the first column. This means, if for the first column there is also a column template defined, this will be overridden.

Group header column template

The DGridGroupHeaderColumnTemplateDirective is rendered in the group header cell of the corresponding column. It is especially useful to show an aggregate value for given columns.

Group footer template

The DGridGroupFooterTemplateDirective is rendered in the group footer. To use the footer template, you have to set the showFooter option of the DGridGroupingSettings configuration object to true.

Example

Group footer column template

The DGridGroupFooterColumnTemplateDirective is rendered in the group footer cell of the corresponding column. It is especially useful to show an aggregate value for specific columns. To use the footer template, you have to set the showFooter option of the DGridGroupingSettings configuration object to true.
