Grouping

To enable grouping just set the groupable property on the grid to true or to DGridGroupingSettings. With this setup a grouping panel appears above the columns and we can drag & drop the column headers on this panel. After dropping the columns we can also reorder them to change the grouping. We can also click on them to sort the groups. If we want to remove grouping by a column, just click on the 'x' icon on the column chip or drag it out of the grouping panel.

DGridGroupingSettings has the following properties:

  • showFooter: true or false, indicating, whether we want a footer for the groups or not
  • aggregates: an array of DGridGroupAggregate - more to read in the 'Aggregation' section
  • stickyHeader: true or false, indicating, whether we want the group headers to be sticky, if we scroll the grid
  • groups: used for predefined grouping settings for the grid
    • field: data field of the given column
    • dir: sort direction
    • compareFn: custom compare function

If we want to define predefined group settings for a colum, we can use the grouping input property on a column, which takes a DGridColumnGroupingSettings object. This object has a mandatory field property, and two optional properties (dir for sort direction and compareFn for a custom comparing function)

Priorities

When grouping, the following applies:

  • If we group by a column and this column has a header template, then it will be rendered at the beginning of the header. Which means, it will override any header template for the first column, if set.
  • If a column has a header template, but also a column header template, and we group by this column, the header template will be rendered.
  • If a column has a footer template or column footer template and we group by this column, then it will override any footer template for the first column, if set.
  • If a column has no footer template or column footer template, but the first column has any footer template, then this footer template will be rendered.
