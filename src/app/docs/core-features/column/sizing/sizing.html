Column sizing

Column width

As described in the Configuration section you can set the width of a column either by setting the width property on the column object in a signal array or by setting the same property on a d-grid-column element.

The following rules apply:

  • When all column widths are explicitly set and the cumulative column width is greater than the available grid width, a horizontal scrollbar appears and all set column widths are respected.
  • When all column widths are explicitly set and the cumulative column width is less than the available grid width, the remaining width is distributed evenly between all columns.
  • When no column widths are set, the available width is distributed evenly between all grid columns, maintaining a minimum width of 50px if not overridden by the minAutoColWidth global input property. If autosize is set, the width of the maximum content (header, body, footer) will be respected.
  • When only some column widths are set and the cumulative width of the columns with set widths is greater than the available grid width, a horizontal scrollbar appears and all set column widths are respected. Columns with no set width are rendered with a defined minimum width 50px if not overridden by the minAutoColWidth global input property. If autosize is set, the width of the maximum content (header, body, footer) will be respected.
  • When only some column widths are set and the cumulative width of columns with set widths is less than the available grid width, the widths of the columns with a set width are respected and the remaining width is distributed evenly between the other columns. If autosize is set, the width of the maximum content (header, body, footer) will be respected.

Column resizing

By default, the column-resizing feature is disabled. To enable column resizing, set the resizable property to true. If the user resizes the Grid columns so that the total width of the columns becomes less than the width of the Grid, the remaining table is filled with whitespace.

To restrict the resizing of the column within a certain range, set its minResizableWidth and maxResizableWidth properties. If the initial width of the given column is greater, than the maximum allowed width, you will notice a little jump when starting resizing, because the column width will be set to the maximum allowed width.

Important: minResizableWidth should be set to at least the sum of the largest horizontal padding (left+right) in any of the grid templates plus the minimum content width of the given cell, which should be visible plus the border (usually 1px) - this ensures, that columns don't overlap if we resize to the very minimum.

Upon double click on the resize element, the column width is aligned to the largest content. In this case minResizableWidth and/or maxResizableWidth aren't respected.

Constrained mode

By default, when the resizable functionality of the grid is enabled, column widths adjust relative to the entire component. If the total columns width exceeds the grid's width, a horizontal scrollbar appears. If the total columns width is less than the grid's width, the remaining space is displayed as white space.

To prevent this behavior and ensure that adjacent columns resize only up to their outer borders, set the resizable property to a variable of type DGridResizeMode with a value equal to constrained. To retain the default behavior, set the resizable property to unconstrained or to true.

In constrained mode we have to stop resizing the second last column, if the last column reaches a certain width. This width is set currently to 40 pixels.
